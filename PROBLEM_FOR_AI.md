# ETRI 라이프로그 수면/감정 예측 — 다른 AI를 위한 현황·벽 정의서 (갱신판)

목표: public log-loss를 더 낮추기(현재 best 0.5931, 1위권 ~0.55). 아래는 **검증된 사실 / 지금 통하는 것 /
이미 막힌 것 / 남은 열린 질문**이다. 같은 길을 반복하지 말고 "열린 질문"을 공략하라.

---

## 1. 과제 / 데이터 / 평가
- 하루(피험자×날짜)마다 7개 이진 타깃의 **확률**을 예측. 평가 = 7타깃 **평균 Log-Loss**(낮을수록 좋음).
- Q1(수면질)·Q2(피로)·Q3(스트레스): 설문, **그 사람 전체기간(train+test) 평균 대비 위/아래**로 이진화.
  S1(총수면)·S2(수면효율)·S3(입면지연)·S4(각성): **Withings 침대센서** 기반 NSF 가이드라인 충족 여부.
- 라벨 train 450일 / test 250일(**라벨 없음**). **피험자 id01~id10 단 10명, train·test 동일인**,
  test 날짜는 각 사람의 **뒤쪽(나중) 블록**(부분적 미래예측). 센서 12종 700일분.
- 공개 LB = test의 44%(약 110행). **하루 3회 제출. 규칙상 LB는 수상순위와 무관(논문이 결정).**

## 2. ★ 검증 방법 (이게 실제 LB와 일치함이 4번 제출로 확인됨 — 반드시 이걸로 판정)
- 무작위 CV 금지(인접일 누수로 OOF 0.58 착시 → 실제 0.60).
- **subject_time_blocked CV**: 피험자별 날짜를 시간순 5블록으로 나눔. 그중 **마지막 블록(가장 나중, 과거만
  학습) 평균 Log-Loss = "last-block 점수"**. 이게 test와 동일 regime.
- 검증된 일치(예측 last-block → 실제 LB): 0.6025→0.6034, 0.5933→0.5931, 0.5957→0.5972.
  **즉 last-block에서 좋아지면 LB도 좋아진다. 제출 안 쓰고 로컬로 판정 가능.**

## 3. 지금 통하는 것 (현재 best, 0.5931) — 여기서 출발하라
센서 LGBM 단독은 0.60 벽. **타깃별로 "센서 모델 + 타깃 자신의 시계열 prior"를 섞으면 0.593까지 내려간다.**
`src/train_temporal_prior.py`의 고정 recipe(인과적·test 실현가능, 모두 멀티시드):

| 타깃 | recipe | last-block | 직관 |
|---|---|---|---|
| Q1 | model(LGBM) | 0.619 | 센서가 그나마 나음 |
| Q2 | 0.3·model + 0.7·최근20일평균 | 0.638 | 예측불가 → base rate로 수축 |
| Q3 | 0.2·model + 0.8·최근2일평균 | 0.635 | 예측불가 → 최근값으로 수축 |
| S1 | model | 0.450 | 센서 강함(잘 맞음) |
| S2 | 0.3·model + 0.7·피험자 시간추세(ridge) | 0.623 | 효율이 기간 따라 드리프트 |
| S3 | 피험자 prior(스무딩평균) | 0.601 | r=0.93, 늘 비슷 → prior가 최고 |
| S4 | 0.5·model + 0.5·시간추세(ridge) | 0.589 | 드리프트 |

핵심 원리 2가지(둘 다 LB로 검증됨):
1. **순방향 수축**: full-OOF로 블렌드 가중을 고르면 센서모델을 과신한다(낙관). 미래(last-block)에선
   예측불가 타깃(Q2/Q3, r≈0)·prior우세 타깃(S3, r=0.93)을 **base rate/prior 쪽으로 수축**해야 이득.
2. **피험자별 선형 시간추세(ridge)**: S2/S4는 기간에 걸쳐 드리프트가 있어, 날짜로 회귀해 미래로 외삽하면 이득.

주의: 마진가드를 건 "보수적 자동선택"(`src/train_final.py`)은 오히려 0.5972로 더 나빴다. recipe의 과감한
선택(Q3 last2 등)이 노이즈가 아니라 진짜였다. 즉 **작은 last-block 신호도 의외로 LB로 이어진다.**

## 4. 타깃별 지속성(같은 사람 전반부↔후반부 비율 상관 r) — 전략의 근거
Q1 r=0.66 · Q2 r=0.01 · Q3 r=0.09 · S1 r=0.62 · S2 r=0.64 · S3 r=0.93 · S4 r=0.61.
→ Q2·Q3는 개인평균이 무의미(그날그날). S3는 거의 고정. 나머지는 개인 경향 유효.

## 5. 이미 시도했고 '효과 없음' 확인 (반복 금지, 전부 검증 CV로)
- 모델: LightGBM(최고) > XGBoost·CatBoost·ExtraTrees·로지스틱·MLP·BiLSTM. 앙상블·캘리브레이션 무효.
- 센서 피처 추가(워치 HRV, GPS 위치엔트로피/이동반경, WiFi'집' 지표, 행동 lag/rolling): 단변량 AUC 높아도
  모델에선 무효(부스팅이 이미 상관신호 추출). 피처 선별(top-K) 악화.
- prior 스무딩·확률클리핑·하이퍼파라미터·멀티타깃 스태킹: 무효.
- **외부 수면데이터 sleep-staging: 입력 막힘** — 워치 야간 심박이 너무 듬성(라벨밤당 시간범위 중앙값 2.0h,
  33%의 밤은 표본 30개 미만, 버스트 기록). 연속 야간 생리신호가 없어 적용 불가.

## 6. 현재 벽 / 열린 질문 (여기를 풀어라)
1. **temporal recipe는 90행 last-block에서 grid로 튜닝됐다.** 더 체계적으로(과적합 없이) 시계열·순방향
   구조에서 추가 신호를 뽑을 방법은? (보수적 자동선택은 0.5972로 후퇴 → 너무 사리면 손해, 너무 짜면 노이즈.)
2. **Q1(r=0.66로 지속적인데 forward에서 0.62)**: 개인 경향이 있는데도 센서모델이 그만큼 못 한다. 피험자별
   forward 모델/캘리브레이션으로 Q1을 더 낮출 수 있나?
3. **멀티태스크/타깃 상관(S2-S4 0.48, Q1-S1 0.36) 미사용.** 7타깃 동시학습·상호 보정으로 강도를 빌릴 수 있나?
4. **Q 타깃 임계값 = 전체기간(train+test) 평균.** 즉 한 피험자의 전체기간에서 Q는 ~50%로 균형. test 라벨은
   train 라벨과 합쳐 그 균형을 맞춰야 한다(제약). 이 구조적 제약을 예측에 활용할 수 있나? (미해결.)
5. **S2/S3/S4(객관 수면)**: 침대센서가 없고 워치 야간이 듬성. 단, **폰 야간 신호는 촘촘하다(화면/충전 야간
   타임스탬프 ~571개)**. 폰 기반으로 효율(S2)·각성(S4)을 더 정밀하게 복원할 여지는?
6. **노이즈 바닥**: 공개 LB 110행이라 동일 모델도 ±0.033 출렁. 0.59→0.58 구분은 신중히(검증 CV로 먼저).

## 7. 산출물 규칙
- `data/`만 입력(외부데이터/사전학습모델은 규칙상 허용, 출처 명기). 인과적·test 실현가능해야 함(test 라벨 없음).
- 반드시 **subject_time_blocked last-block**으로 자체 검증하고 0.593 미만임을 보여라. 무작위 CV의 OOF(0.58)를
  성과로 착각 금지.

## 8. 코드 위치
- `src/train_temporal_prior.py`: 현재 best recipe(0.5931). `src/train.py`: 센서 LGBM 베이스(0.6034).
- `src/train_final.py`: 통합·자동선택 시도(0.5972, 참고용). `src/evaluate.py`: 빠른 last-block 평가.
- `src/eda.py`: full vs last AUC로 진짜/가짜 신호 구분 시각화. `EXPERIMENTS.md`·메모리에 실패 목록 정리.
- 환경: Python, pandas/lightgbm/sklearn. GPU 불필요(CPU 수분).
